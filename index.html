Protocolo SPIRIT

CONSORT vs. SPIRIT

CONSORT

  • Consolidated Standards of Reporting Trials
  • Para relatar resultados de um RCT já concluído
  • Checklist de 25 itens para artigo de resultados
  • Garante transparência no relato

SPIRIT

  • Standard Protocol Items: Recommendations for Interventional Trials
  • Para planear e registar um RCT antes de começar
  • Checklist de 33 itens para protocolo
  • Garante rigor no planeamento

Chan et al., BMJ 2024

CONSORT = o que aconteceu. SPIRIT = o que vai acontecer.

A distinção é importante: os estudantes vão escrever um protocolo SPIRIT — um documento que existe ANTES de qualquer dado ser recolhido. CONSORT é o que usarão quando (e se) publicarem os resultados. O link aponta para a versão actualizada de 2024 do SPIRIT.

Usar IA no Protocolo

Porquê Usar IA?

A IA é útil para:

  • Gerar um rascunho inicial de uma secção a partir das vossas ideias
  • Sugerir estrutura de frases e formulações técnicas
  • Expandir pontos que escreveram em lista para texto corrido
  • Identificar inconsistências ou lacunas no que escreveram

A IA não substitui:

  • O vosso conhecimento do cenário
  • A verificação de factos e números
  • A consistência com a questão PICO
  • O julgamento clínico e metodológico

Este ponto é importante: a IA é um acelerador de redação, não um substituto de pensamento. Um rascunho gerado por IA sem revisão crítica tem erros factuais, critérios vagos e inconsistências internas — é previsível.

Workflow com IA

1. Preparem o contexto
   → Cenário, PICO, secção a escrever, instruções do template

2. Escrevam um prompt claro
   → O que querem, em que formato, com que restrições

3. Gerem o rascunho
   → ChatGPT, Copilot, Claude, ou outro

4. Revejam criticamente
   → Factos, especificidade, consistência com PICO

5. Editem e integrem
   → Adaptem ao Markdown, corrijam, completem

O passo 4 é o mais importante e o mais frequentemente saltado. É também o que mais distingue uso competente de uso ingénuo da IA.

Exemplo de Prompt — Racional

Estou a escrever o racional de um protocolo SPIRIT para um ensaio clínico randomizado.

Cenário: App móvel CardioFit de reabilitação cardíaca para sobreviventes de enfarte agudo do miocárdio (≥ 18 anos, FEVE ≥ 40%).
Comparador: Reabilitação cardíaca convencional isolada.
Outcome primário: Capacidade funcional (VO₂ máx) às 12 semanas.

Escreve um racional com ~350 palavras, estruturado em 4 parágrafos: 1. Magnitude da doença cardiovascular em Portugal e Europa (dados epidemiológicos específicos) 2. Limitações da reabilitação cardíaca convencional (adesão, acessibilidade, abandono) 3. Evidência sobre apps móveis em reabilitação cardíaca (citar mecanismos de ação) 4. Gap específico que este estudo preenche e justificação do comparador

Usar linguagem científica formal. Não usar marcadores de lista — texto corrido.

Mostrar este prompt ao vivo e deixar a IA gerar. Depois comentar o resultado: o que está bem, o que precisaria de verificação (especialmente números epidemiológicos e valores de VO₂), o que está genérico de mais.

Revisão Crítica — O Que Verificar

Depois de gerar, verificar sempre:

  • Números e estatísticas — confirmar em fontes (PubMed, OMS, SPP)
  • Especificidade dos critérios — ❌ “adultos” → ✅ “18–65 anos, HbA1c 7–9,5%”
  • Consistência com PICO — os critérios correspondem à população definida?
  • Descrição da intervenção — é a vossa app, não uma app genérica
  • Terminologia técnica — GINA, PHQ-9, ACT, HbA1c — usada correctamente?
  • Nada copiado sem compreender — se não conseguem explicar, reescrevam

Um exercício útil: pedir a um colega de grupo que leia uma secção e aponte o que não entende ou parece vago. Se não conseguem defender o que está escrito, é sinal de que precisam de rever.

Conversão com Pandoc

O Que É o Pandoc?

Conversor universal de documentos

  • Markdown → Word, PDF, HTML, LaTeX, …
  • Preserva estrutura: títulos, tabelas, listas
  • Suporta metadados YAML
  • Gratuito e open source

Formatação Profissional

Microsoft Word

Porquê Usar Estilos?

Sem estilos:

  • Formatar cada título manualmente
  • Inconsistências inevitáveis
  • Índice impossível de criar automaticamente
  • Mudança global = refazer tudo

Com estilos:

  • Consistência automática
  • Índice em 3 cliques
  • Mudança global instantânea
  • Navegação fácil no documento

Regra: Nunca formatar títulos manualmente. Sempre via estilos.

Como Aplicar Estilos

  1. Seleccionar título principal → aplicar “Título 1”
  2. Seleccionar subtítulos → aplicar “Título 2”
  3. Subsecções → aplicar “Título 3”
  4. Texto normal → “Normal” (geralmente já está)

Modificar estilos

Click direito no estilo → Modificar → alterar fonte, tamanho, cor.
A alteração aplica-se automaticamente a todos os títulos com esse estilo.

Referências Bibliográficas

Porquê Gerir Referências?

O problema:

“Jiang et al., 2023 mostrou que…” — qual Jiang? Qual artigo?

A solução: citation keys

@jiang2023mhealth    →  Jiang et al. (2023), mHealth...
@who2022asthma       →  WHO (2022), Global Asthma Report

No vosso protocolo Markdown:

Apps móveis aumentam a adesão em 35% [@jiang2023mhealth].

→ Pandoc converte automaticamente para qualquer estilo (Vancouver, APA…)

Este é o conceito central: o citation key é um identificador estável que liga o texto à referência no ficheiro .bib. A IA pode sugerir referências, mas cada uma tem de ser verificada e importada para o Zotero manualmente.

O Workflow de Referências

 Zotero          Better BibTeX         LaTeX / Pandoc
(biblioteca) ──▶ (citation keys) ──▶  (documento final)
                   exporta .bib
  • Zotero: gestor de referências (gratuito, open source)
  • Better BibTeX: plugin que gera citation keys estáveis e exporta .bib
  • .bib: ficheiro de texto com todas as referências

O ficheiro .bib é o elo de ligação entre o Zotero e qualquer ferramenta de escrita — Pandoc, LaTeX, Quarto. Na próxima aula vão produzir este ficheiro e usá-lo em LaTeX.
